# Aula 05 - Projeto Integrador

## Criação do repositório
A criação do repositório é o início do projeto no GitHub, onde os arquivos ficam armazenados online.

## Clone do repositório
O clone é a cópia do repositório para o computador, permitindo editar localmente.

## Criação de branch
A branch é uma ramificação usada para desenvolver sem alterar o projeto principal.

## Registro de commits
O commit registra as alterações feitas nos arquivos.

## Envio das alterações (push)
O push envia as mudanças do computador para o GitHub.

## Atualização do repositório local (pull)
O pull atualiza o projeto local com mudanças do repositório online.

## Criação de Issues
Issues servem para registrar tarefas ou problemas.

## Organização com Milestone
Milestones organizam etapas e objetivos do projeto.

## Gerenciamento com Project
Projects ajudam a organizar tarefas em quadros.

## Abertura de Pull Request
O Pull Request solicita a inclusão das alterações no projeto principal.

## Realização de Merge
O Merge junta as alterações ao projeto principal.

## Atualização
Alteração feita na branch desenvolvimento.
